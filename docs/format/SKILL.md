# 文档格式技能

遵循 [量潮科技文档格式标准](https://github.com/quanttide/quanttide-specification-of-business-entity/blob/v0.1.1/docs/format.md)

## 写作原则

- **删**：删除不必要的格式元素，优先用段落和标题
- **简**：能用列表就不表格，能用文字就不列表
- **少**：全文格式元素（分隔线、表格、加粗）尽量少
- **一**：同一概念全程使用相同名称

## 标题规范

- 最多使用三级标题（`#` / `##` / `###`）
- 一级标题仅文档标题使用一次
- 避免在标题中使用标点符号

## 列表规范

无序列表（`-`）适用于并列要点：
- 并列的多个要点
- 不分先后顺序的内容

有序列表（`1.`）适用于有明确顺序的步骤：
1. 有明确顺序的步骤
2. 需要编号的操作流程

## 代码块

必须标注语言类型：

```bash
git status
```

```python
def hello():
    print("Hello")
```

## 表格规范

| 列1 | 列2 | 列3 |
|:----|:---:|----:|
| 左对齐 | 居中 | 右对齐 |

表格用于呈现多维度需对比的数据，避免简单名词-定义对应（用列表代替）。

## 引号规范

中文文档使用中文引号：
- 直接引用：「这是引用内容」
- 术语引用：「变量名」

替代方案：
- 普通强调使用加粗：**关键信息**
- 术语使用行内代码：`variable`
